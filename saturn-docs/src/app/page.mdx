# Welcome to Saturn Documentation {{ id: 'introduction' }}

Saturn is an open-source conversational interface that democratizes cloud operations. We specialize in translating natural language into precise cloud API calls, making cloud infrastructure management accessible to everyone. This documentation will guide you through using the Saturn tool and running ingestion processes. {{ className: 'lead' }}

## Project Requirements {{ id: 'requirements', anchor: true }}

To successfully use the Saturn tool and its associated processes, please ensure you have the following prerequisites:

- Python 3.9 or higher
- Google Cloud SDK (for GCP operations)
- API keys for your preferred LLM provider (OpenAI, Google Gemini, Anthropic Claude, or Mistral)
- If using the RAG engine, the following Python packages are also required:
  - `llama-index>=0.10.0`
  - `llama-index-embeddings-huggingface>=0.1.0`

<p className="mb-6">
Please also refer to the <code>requirements.txt</code> file in the main Saturn project for a full list of Python package dependencies.
</p>

## Installation {{ id: 'installation', anchor: true }}

<p className="mb-2">First, clone the repository and install the necessary dependencies:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    git clone https://github.com/vidhra/Saturn.git<br />
    cd Saturn<br />
    python -m venv venv<br />
    source venv/bin/activate  # On Windows use: .\venv\Scripts\activate<br />
    pip install -e .
  </code>
</pre>

## Configuration {{ id: 'configuration', anchor: true }}

<p className="mb-2">Next, configure your environment. Copy the example configuration file:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    cp config.yaml.example config.yaml
  </code>
</pre>
<p className="mb-2">Then, create a <code>.env</code> file in the project root and add your API keys and GCP Project ID. For example:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    OPENAI_API_KEY=your-openai-api-key<br />
    GCP_PROJECT_ID=your-gcp-project-id
  </code>
</pre>
<p className="mb-2">You can also configure the LLM provider (e.g., <code>openai</code>, <code>gemini</code>, <code>claude</code>, <code>mistral</code>) and other settings in the <code>config.yaml</code> file.</p>
<p className="mb-2">Authenticate with Google Cloud using the gcloud CLI:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    gcloud auth application-default login
  </code>
</pre>

## Basic Usage {{ id: 'basic-usage', anchor: true }}

<p className="mb-2">Once set up, you can run commands using natural language:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    saturn run "List all my compute instances"
  </code>
</pre>
<p className="mb-2">Or, for more complex operations:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    saturn run "Create a serverless VPC access connector named &apos;dev-connector&apos; in us-central1 with IP range 10.9.0.0/28 in the default network.&quot;
  </code>
</pre>

## Running Ingestion (RAG Engine) {{ id: 'rag-engine', anchor: true }}

<p className="mb-4">
Saturn includes a RAG (Retrieval Augmented Generation) engine to enhance command accuracy using cloud documentation. Here&apos;s how to set it up and use it:
</p>

<h3 className="text-xl font-semibold mb-2">1. Install RAG Dependencies</h3>
<p className="mb-2">If you haven&apos;t already, install the required packages for the RAG engine:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    pip install llama-index&gt;=0.10.0 llama-index-embeddings-huggingface&gt;=0.1.0
  </code>
</pre>

<h3 className="text-xl font-semibold mb-2">2. Ingest Documentation</h3>
<p className="mb-2">Use the <code>saturn ingest-docs</code> command to build your knowledge base. You&apos;ll need to specify a vector store and the path to your documentation files. Saturn currently supports ChromaDB, DuckDB, and In-Memory vector stores.</p>
<p className="mb-2">Example using ChromaDB and local documentation:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    saturn ingest-docs --vector-store chroma --docs-path ./internal/tools/gcloud_online_docs_markdown
  </code>
</pre>
<p className="mb-2">
  Make sure the <code>--docs-path</code> points to the location of the markdown files for the cloud documentation you want to ingest.
</p>

<h3 className="text-xl font-semibold mb-2">3. Run Commands with RAG Enhancement</h3>
<p className="mb-2">To use the RAG engine when running commands, specify the vector store:</p>
<pre className="bg-gray-900 text-white p-3 rounded-md text-sm overflow-x-auto mb-4">
  <code>
    saturn run "Create a secure GKE cluster with private nodes" --vector-store chroma
  </code>
</pre>
<p className="mb-2">
  This tells Saturn to use the knowledge base you ingested for more accurate command generation.
</p>
